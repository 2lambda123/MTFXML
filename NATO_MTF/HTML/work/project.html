MTF XML Refactoring Project
Major James D. Neushul, USMC
Goals:
1. Adopt the "Garden of Eden" Schema design.
2. Make MTF XML more readable, re-usable and useful.
3. Encourage web implementations for message generation, parsing, validation and use.
4. Encourage web implementations for configuration management.
5. Effectively leverage the ontology for imnproved Coalition interoperability and combat effectiveness.
6. Leverage USMTF Alignment with U.S. National Information Exchange Model (NIEM)
Basic Approach:
This work follows the proposals from an XML-MTF brief provided by AFC2IC/C2IIM. The conceptual model described employs schemas for messages, segments, sets, fields and datatypes. This work has generated the following integrated schemas based on examples provided by Mr. Jacob Satterfield.
Current XML MTF Baseline Schemas:
fields.xsd - 24.4 MB
composites.xsd - 1.0 MB
sets.xsd - 34.1 MB
messages.xsd - 16.7 MB
Generated NATO MTF GoE SCHEMAS:
mtf_fields.xsd - 1.5 MB
mtf_composites.xsd - 13.9 MB
mtf_sets.xsd - 635 kB
mtf_segments.xsd - 635 kB
mtf_messages.xsd - 23.5 MB
